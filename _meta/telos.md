# Telos — why llm-bench exists

To convert the **unknowable, faster-than-comprehension model ecosystem** into **earned,
evidence-backed operational trust** — so Aria can run a multi-model agent ensemble and route
each piece of real work to the model that has *earned* that responsibility, instead of
defaulting to whatever is familiar.

The repository began as a benchmark (scores on standardized tasks). As of 2026-06-27 it is a
**Model Familiarity Engine** (see `VISION.md`). The name stays for its history; the purpose
is now onboarding, not grading.

**The question this project answers** is not *"how good is this model?"* but
*"what responsibilities has this model earned, on the work I actually do?"*

**It exists because** model choice is currently hearsay, and hearsay does not scale to a
weekly flood of new models or to an ensemble where the wrong seat for a model is a silent,
compounding cost. Replace opinion with observation.

**It succeeds when** a new model can be dropped in, observed in real work, and placed —
plan / implement / review / brainstorm / critique / never-touch-production — with evidence
and a confidence level behind every placement, and when that placement keeps updating as the
model and the work change.

If this ever drifts back toward producing a leaderboard number, it has lost its telos.
